# Contributing to GEO Checklist

Contributions are welcome. This checklist improves when practitioners share what they learn from real GEO audits.

## How to Contribute

**Suggest a new checklist item:**
Open an issue describing the domain (Schema, llms.txt, Entity Consistency, etc.), the proposed item, and why it matters.

**Submit a correction:**
Open a pull request with the fix. If it's a factual correction, include a source.

**Add an industry example:**
Create a new file in `examples/` following the format of existing examples. Submit a pull request.

## Guidelines

- Keep items actionable. Each checklist item should be something a practitioner can verify or implement.
- Include verification methods. How does someone confirm the item is done?
- Avoid marketing language. This is a technical resource.
- Provide evidence when possible. Link to documentation, research, or case studies.
- Write in English for global accessibility.

## Code of Conduct

Be professional and constructive. We're here to build a useful resource for the GEO community.
